# Sign-Lang Project

This repository contains the notebook workflow for preparing ASL and SGSL video data, extracting MediaPipe pose and hand keypoints, and training sequence models for sign-language classification.

## Folder Navigation

```text
.
├── 01_asl_video_augmentation.ipynb
├── 02_asl_normalize_60_frames.ipynb
├── 03_mediapipe_pose_hands_videos.ipynb
├── 04_train_lstm_asl_sgsl_keypoints.ipynb
├── 05_train_stgcn_asl_sgsl_keypoints.ipynb
├── Videos/                         # Local dataset folder, ignored by git
├── artifacts_lstm_asl_sgsl/         # LSTM model checkpoints and confusion matrices
├── artifacts_stgcn_asl_sgsl/        # ST-GCN checkpoints and confusion matrices
└── Papers/                          # Reference papers used for the project
```

The `Videos/` folder is intentionally excluded from git because the raw and generated videos are large. Keep this folder locally in the project root when running the notebooks.

## Expected Video Folder Layout

Place the source videos under `Videos/` using one folder per dataset and one subfolder per class.

```text
Videos/
├── ASL/
│   ├── africa/
│   │   ├── 0.mp4
│   │   ├── 1.mp4
│   │   └── ...
│   └── <asl_class>/
│       └── ...
└── SGSL/
    ├── <sgsl_class>/
    │   ├── 0.mp4
    │   ├── 1.mp4
    │   └── ...
    └── ...
```

Generated folders are also written under `Videos/`:

```text
Videos/
├── ASL_60/                          # ASL clips normalized to 60 frames
├── ASL_60_pose_hands_video/         # ASL overlay videos with pose and hand landmarks
├── ASL_60_pose_hands_keypoints/     # ASL frame-level keypoint .npy files
├── SGSL_pose_hands_video/           # SGSL overlay videos with pose and hand landmarks
└── SGSL_pose_hands_keypoints/       # SGSL frame-level keypoint .npy files
```

If `Videos/ASL_60` does not exist, the MediaPipe notebook can fall back to `Videos/ASL` and write `ASL_pose_hands_video/` and `ASL_pose_hands_keypoints/` instead.

## Notebook Workflow

Run the notebooks in numeric order unless you already have the corresponding generated data.

### `01_asl_video_augmentation.ipynb`

Augments the original ASL videos in `Videos/ASL/<class>/` until each ASL class has 30 clips. It applies random spatial and appearance transforms, then writes additional `.mp4` files back into each class folder.

This notebook also includes video inspection cells for ASL and SGSL frame sizes, frame counts, and resolutions.

### `02_asl_normalize_60_frames.ipynb`

Normalizes ASL clips to a fixed 60-frame length so ASL matches the SGSL sequence length used later in the pipeline.

Input:

```text
Videos/ASL/<class>/*.mp4
```

Output:

```text
Videos/ASL_60/<class>/*.mp4
```

The notebook includes verification cells to confirm that all normalized ASL clips contain exactly 60 frames and to compare frame counts with `Videos/SGSL`.

### `03_mediapipe_pose_hands_videos.ipynb`

Runs MediaPipe Pose Landmarker and Hand Landmarker on ASL and SGSL videos. It creates two kinds of outputs:

- Overlay videos showing detected pose and hand landmarks.
- Per-frame keypoint arrays saved as `.npy` files.

Inputs:

```text
Videos/ASL_60/<class>/*.mp4
Videos/SGSL/<class>/*.mp4
```

Outputs:

```text
Videos/ASL_60_pose_hands_video/<class>/<clip>.mp4
Videos/ASL_60_pose_hands_keypoints/<class>/<clip>/frame_XXXXXX.npy
Videos/SGSL_pose_hands_video/<class>/<clip>.mp4
Videos/SGSL_pose_hands_keypoints/<class>/<clip>/frame_XXXXXX.npy
```

Each keypoint frame is saved with shape `(75, 5)`, where the five columns are `x`, `y`, `z`, `visibility`, and `presence`. Rows `0-32` are pose landmarks, rows `33-53` are left-hand landmarks, and rows `54-74` are right-hand landmarks.

### `04_train_lstm_asl_sgsl_keypoints.ipynb`

Trains separate LSTM models for ASL-only and SGSL-only classification using the keypoint folders generated by notebook `03`.

Inputs:

```text
Videos/ASL_60_pose_hands_keypoints/
Videos/SGSL_pose_hands_keypoints/
```

Outputs:

```text
artifacts_lstm_asl_sgsl/
├── lstm_ASL_only.keras
├── lstm_SGSL_only.keras
├── <run>_<split>_cm.csv
├── <run>_<split>_cm_norm.csv
└── <run>_<split>_cm.png
```

The notebook uses an 80/10/10 train/validation/test split and exports confusion matrices for train, validation, and test sets.

### `05_train_stgcn_asl_sgsl_keypoints.ipynb`

Trains separate ST-GCN models for ASL-only and SGSL-only classification using the same MediaPipe keypoint layout as notebook `04`.

Inputs:

```text
Videos/ASL_60_pose_hands_keypoints/
Videos/SGSL_pose_hands_keypoints/
```

Outputs:

```text
artifacts_stgcn_asl_sgsl/
├── stgcn_ASL_only_best.pt
├── stgcn_SGSL_only_best.pt
├── <run>_<split>_cm.csv
├── <run>_<split>_cm_norm.csv
└── <run>_<split>_cm.png
```

The ST-GCN graph uses 75 nodes: MediaPipe body pose landmarks, left-hand landmarks, and right-hand landmarks. The model connects pose landmarks, hand landmarks, and wrist-to-hand bridges, then evaluates with the same train/validation/test structure as the LSTM notebook.

## Recommended Run Order

1. Add raw source videos under `Videos/ASL` and `Videos/SGSL`.
2. Run `01_asl_video_augmentation.ipynb` to balance ASL classes.
3. Run `02_asl_normalize_60_frames.ipynb` to create `Videos/ASL_60`.
4. Run `03_mediapipe_pose_hands_videos.ipynb` to generate overlay videos and keypoint arrays.
5. Run `04_train_lstm_asl_sgsl_keypoints.ipynb` for LSTM experiments.
6. Run `05_train_stgcn_asl_sgsl_keypoints.ipynb` for ST-GCN experiments.

## Notes

- The generated `Videos/` data is local only and is not committed to git.
- Model artifacts and confusion matrices are stored in `artifacts_lstm_asl_sgsl/` and `artifacts_stgcn_asl_sgsl/`.
- Keep the class-folder names consistent across source videos and generated keypoint folders because the training notebooks use folder names as labels.
